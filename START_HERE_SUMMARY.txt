## 🚀 Getting Started - 3 Steps

### 1. Start Application
```bash
python -m uvicorn app.main:app --reload
```

### 2. Register & Login
```bash
# Register
curl -X POST "http://localhost:8000/api/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"username":"user","email":"user@example.com","full_name":"User","password":"pass123"}'

# Login  
curl -X POST "http://localhost:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"user","password":"pass123"}'
```

### 3. Use Token
```bash
curl -X GET "http://localhost:8000/api/auth/me" \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

## 🔐 Security Notes

### ⚠️ CRITICAL - Before Production
**Change the SECRET_KEY in `app/auth.py` line 15**

Current:
```python
SECRET_KEY = "your-secret-key-change-in-production"
```

Generate new key:
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

Update to:
```python
SECRET_KEY = os.getenv("AUTH_SECRET_KEY", "your-generated-key")
```

## 🚨 Common First-Time Issues

| Issue | Solution |
|-------|----------|
| "ModuleNotFoundError: passlib" | Run: `pip install -r requirements.txt` |
| "ModuleNotFoundError: app.auth" | Run from workspace root: `cd g:\PLCM` |
| "401 Unauthorized" | Add token to header: `Authorization: Bearer <token>` |
| "403 Forbidden" | User doesn't have permission - assign role via admin endpoint |
| "Database error" | Set DATABASE_URL in .env file |
| "Token invalid" | Token may have expired - re-login to get new one |
